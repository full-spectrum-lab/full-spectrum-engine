# Local-First Engine

## One-sentence positioning

`full-spectrum-engine` is the local-first engine package of the Full Spectrum ecosystem.

It allows an enterprise, team, researcher, or individual to run a minimum governance simulation locally for AI decisions, customer-service conflicts, knowledge-source conflicts, and multi-agent coordination scenarios.

## It does not require protocol-network entry first

Using this engine does not require:

- registering into a Full Spectrum protocol network;
- uploading enterprise data;
- connecting to external guardian nodes;
- using a centralized service;
- exposing internal logs;
- claiming any external protocol certification.

An enterprise can use it purely as an internal governance tool.

## Relationship to the cell-protocol layer

This engine answers:

> How is an event observed, simulated, compressed into risk, and recorded into an audit path?

The cell-protocol layer answers:

> Who is the subject, what may it do, what may it not do, who is responsible, and when should it exit or downgrade?

So:

- the engine layer can run independently;
- the cell-protocol layer strengthens identity, permission, and responsibility anchoring;
- different enterprises may start with only the engine layer and introduce the cell layer later.

## Relationship to the protocol-network layer

The protocol-network layer is for cross-subject coordination, such as:

- cross-enterprise audit exchange;
- external node certification;
- distributed ESS review;
- guardian review;
- contribution and frequency-economy records.

The current engine does not force that step.

Recommended progression:

```text
Phase 1: deploy the local engine internally
Phase 2: formalize cell declarations when internal governance gets more complex
Phase 3: join protocol-network functions only when cross-subject interaction becomes necessary
```

## 中文说明

这份引擎最核心的意义，不是先把企业拉进一个外部网络，而是先让每个主体在自己的边界内：

- 看见风险；
- 留下审计记录；
- 训练自己的判断；
- 再决定是否要进入更重的协议协作层。
