## 1. Common response headers

| Header | Meaning |
| --- | --- |
| `X-Storage-Mode` | Current persistence mode. At `v0.9.0a1` this is `sqlite-persistent`. |
| `X-Full-Spectrum-Notice` | Public boundary notice. Current value: `local-dev-only`. |
| `X-Production-Ready` | Explicit production boundary flag. Current value: `false`. |
| `X-Decision-Id` | Deterministic decision record ID generated by `POST /evaluate`. |
| `X-Input-Metrics-Persisted` | Whether adapter input metrics were intentionally preserved in the result path. |

These headers are deliberately kept out of the main response body so the body can stay close to CLI output.

## 2. `POST /api/v1/evaluate`

This is the main inspection endpoint.

It supports two request modes.

### Mode A: direct scenario

| Field | Type | Required | Meaning |
| --- | --- | --- | --- |
| `scenario` | object | yes | Full scenario config equivalent to `simulate.py --config`. |
| `seed` | integer | no | Deterministic seed. Default `42`. |

### Mode B: adapter mode

| Field | Type | Required | Meaning |
| --- | --- | --- | --- |
| `industry` | string | yes | Registered adapter name such as `ecommerce_customer_service`. |
| `metrics` | object | yes | Normalized business metrics consumed by the adapter. |
| `seed` | integer | no | Deterministic seed. Default `42`. |
| `include_input_metrics` | boolean | no | Whether raw adapter input metrics should be included. Default `false`. |
| `simulation_id` | string | no | Scenario identifier for this evaluation. |
| `input_query` | string | no | Human-readable description of the inspected request. |
| `sensitivity_level` | string | no | `low`, `medium`, or `high`. |
| `enterprise_id` | string | no | Enterprise context identifier. |
| `rule_version` | string | no | Rule-set version label. |

### Evaluate response body: key top-level fields

| Field | Meaning |
| --- | --- |
| `simulation_id` | Stable business-facing scenario label. |
| `timestamp` | Deterministic or generated simulation timestamp. |
| `input_query` | The inspected request or dispute statement. |
| `initial_state` | Governance starting point before simulation. |
| `final_state` | Governance state after simulation. |
| `fshi` | Full Spectrum Health Index summary. |
| `ess` | ESS-lite path exploration result. |
| `validation` | Validation outcome for this run. |
| `risk_vector` | The current engine-layer structured risk summary. |
| `runestone` | Audit token emitted from the decision path. |
| `causal_chain` | Human-readable explanation chain of why the path was produced. |

## 3. `RiskVector`

`RiskVector` is the compact explainability core.

| Field | Meaning |
| --- | --- |
| `survival_impact` | Impact on basic system continuity / loss containment. |
| `trust_impact` | Impact on relationship stability, trust, escalation, or complaint risk. |
| `meaning_impact` | Impact on legitimacy, narrative coherence, or human acceptance. |
| `reversibility` | Current field name retained for compatibility; semantically this behaves closer to **irreversibility / rollback difficulty**. |
| `explainability` | How interpretable the current decision path is. |
| `diffusivity` | How likely the issue is to spread across users, channels, or workflows. |
| `urgency` | How quickly intervention is needed. |
| `uncertainty` | How incomplete or unstable the current evidence is. |

## 4. `POST /api/v1/runestone`

This endpoint creates a standalone audit token without running a full simulation.

| Field | Type | Required | Meaning |
| --- | --- | --- | --- |
| `decision` | string | yes | Decision code such as `W3`. |
| `reason` | object | yes | Must include `enterprise_id` and `rule_version`. |
| `risk_vector` | object | yes | Full eight-dimensional risk vector. |
| `parent_runestone` | string | no | Previous runestone in an audit chain. |
| `agent_trail` | array | no | Participating agents or roles. |
| `ess_snapshot` | object | no | Optional ESS context snapshot. |
| `seed` | integer | no | Used for deterministic runestone generation. |

## 5. Audit list endpoints

### `GET /api/v1/audit/decisions`

| Param | Meaning |
| --- | --- |
| `limit` | Page size, 1 to 100. |
| `offset` | Pagination offset. |
| `adapter` | Optional adapter filter. |
| `since` | Optional ISO 8601 lower bound. |

### `GET /api/v1/audit/runestones`

| Param | Meaning |
| --- | --- |
| `limit` | Page size, 1 to 100. |
| `offset` | Pagination offset. |
| `since` | Optional ISO 8601 lower bound. |

### `DELETE /api/v1/audit/decisions`

| Param | Meaning |
| --- | --- |
| `confirm=true` | Mandatory safety valve. |
| `before=<ISO8601>` | Delete records before a timestamp. |
| `all=true` | Delete all records. Only allowed on local bind. |

## 6. Structured error codes

At `v0.9.0a1`, API error output is standardized into:

```json
{
  "detail": {
    "message": "human-readable explanation",
    "error_code": "VALIDATION_ERROR"
  }
}
```

Current public error codes:

| Error code | Typical meaning |
| --- | --- |
| `VALIDATION_ERROR` | Request shape, field type, pagination, or required-field problem. |
| `ADAPTER_NOT_FOUND` | The requested adapter is not registered. |
| `SIMULATION_ERROR` | Scenario can be parsed but simulation logic rejects it. |
| `INTERNAL_ERROR` | Unexpected internal runtime failure. |
| `STORAGE_ERROR` | SQLite write or persistence failure. |
| `NOT_FOUND` | Requested decision or runestone does not exist. |
| `FORBIDDEN` | Operation blocked by explicit boundary, such as non-local delete. |

External callers should key their program logic off `error_code`, not fragile message text matching.

## 7. Minimal contract interpretation

The API contract at this stage means:

- the request modes are explicit
- the response shape is stable enough to review
- audit artifacts are queryable locally
- error handling is no longer ad hoc

It does **not** mean:

- final enterprise action is automated
- legal responsibility is transferred to the engine
- cross-node protocol interoperability is complete
