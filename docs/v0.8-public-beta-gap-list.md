# v0.8 Public Beta Gap List

> Date: 2026-07-10  
> Scope: first external-facing public beta closure for `full-spectrum-engine`

## Goal

Turn the engine from a strong internal alpha into a public beta package that feels:

- runnable
- reproducible
- explainable
- bounded

This document records the closure checklist used for the v0.8 line.

---

## Already true at the start of v0.8 closure

The engine already had a better technical base than a typical early preview:

- local-first simulation flow works
- `simulate.py --seed` already supports deterministic output
- REST API layer already exists
- SQLite persistence already exists
- test suite already passes locally
- `RiskVector` / `Runestone` / causal-chain outputs already form a coherent explainability chain

So v0.8 is not a rewrite. It is a **public-facing closure pass**.

---

## Gap items

### 1. Public-facing version identity

**Gap**  
The repository still presented itself mainly as `v0.7.2-alpha release-quality cleanup`.

**Closure target**  
Promote the repo framing to `v0.8.0-beta (Public Beta)` while keeping architecture boundaries stable.

**Status**  
Closed in this cycle.

---

### 2. First-run readability

**Gap**  
A new reader could run the project, but the public entry path still felt like an internal engineering repo.

**Closure target**

- better README opening
- explicit quick start
- explicit validation command
- explicit reading order

**Status**  
Closed in this cycle.

---

### 3. Reproducibility beyond “seed exists”

**Gap**  
The code supported deterministic runs, but public beta needed stronger proof than “there is a seed flag”.

**Closure target**

- golden samples committed into the repo
- automated validation script
- tests that compare runtime output to golden samples

**Status**  
Closed in this cycle.

---

### 4. Public boundary clarity

**Gap**  
The repo needed a cleaner answer to “what this engine does not do yet”.

**Closure target**

- make the public preview boundary easy to find
- keep four-layer architecture / Cell Manifest / protocol network clearly out of current scope

**Status**  
Closed in this cycle.

---

### 5. Cross-repo ecosystem clarity

**Gap**  
Older links and repo names still pointed to the previous public structure.

**Closure target**

- align links to `full-spectrum-lab`
- separate protocol / engine / commons / qpp roles more clearly

**Status**  
Closed in this cycle for engine-facing docs.

---

## Explicit non-goals for v0.8

This closure does **not** attempt to finish:

- full four-layer recursive architecture
- Cell Manifest
- protocol-network interoperability
- organization-level aggregation
- complete DreamBrain
- Frequency Economy settlement
- production-grade enterprise auth / access control
- enterprise final action execution

Those belong to later lines and should not be force-fitted into the public beta.

---

## Release standard for calling it “public beta”

v0.8 can be treated as a public beta when all of the following are true:

1. `python -m pytest tests -v` passes locally
2. deterministic samples validate against committed golden outputs
3. README is enough for an external technical reader to run a first scenario
4. non-goals are stated clearly
5. GitHub public mirror can present the same runnable package as Gitee development source

---

## Recommended next step after v0.8

After this closure, the next meaningful line is not “add everything”.

The next disciplined move should be:

1. keep collecting external beta feedback
2. harden the API / sample / report surface
3. prepare a narrower v0.9 or v1.0-beta scope around stable engine contracts
4. delay heavy four-layer / network / manifest expansion until public beta feedback is digested
